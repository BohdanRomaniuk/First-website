Розробити веб-застосування трьохшарової архітектури (із збереженням усіх правил цієї архітектури):

1) клієнтаська частина: html, css, js, ajax (JSON для контенту POST запитів), 
також можуть застосовуватись: js-фреймворки, MVC та ін. технології при умові їх розуміння, а не просто "налаштування".
2) серверна частина: nginx для статичного контенту, nodejs-server (або інший) - для динамічного контенту,
3) сервер бази даних - на вибір (інформаційна модель бд, має бути подана у візуальному вигляді)

для забезпечення роботи інтернет магазину. 

Інтернет магазин повинен забезпечувати наступний функціонал:
1) Авторизація користувачів
2) Адміністративна частина для додавання товарів 
3) (+бали) Чат для переписки з помічником на сайті (напр. викор. websocket)
4) Відображення товарів у зручному для користувача вигляді (потрібно зважати, що товарів може бути велика кількість)
5) Можливість оперативно додавати/скасовувати акції магазину
6) (+бали) Реалізація простого балансувальника навантаження (розподілення навантаження між кількома серверами застосувань)
7) Наявність корзини покупок та кабінету користувача. (історія покупок, відмов і т.д.)
?? (+бали) Розсилка email повідомлень 
(налаштовується в адміністративній частині сайту - реклама, оголошення, акції, благодійність та ін.)
 (наприклад, через google SMTP server).
 
 (+бали) - означає, що даний пункт не є обов"язковий, але принесе додаткові бали у випадку реалізації, 
 конкретна кількість балів залежить від якості реалізації.
 
Тематика магазину та код програм повинні бути унікальні. 
Студент повинен розуміти технології, які застосовував, та пояснити чому обрав саме їх, 
які інші варіанти він розглядав.

Інші деталі завдання та питання - обговорені на заняттях.
